適応度関数

rl-anything には8種類の組み込み fitness が含まれています。--fitness <name> フラグで使い分けます。プロジェクト固有の fitness は scripts/rl/fitness/<name>.py に配置することで追加できます。

🤖 default

LLM 汎用評価。特定ドメインに依存しない汎用的な品質スコアを返す。新規プロジェクトのベースライン計測に最適。

📐 skill_quality

ルールベースの構造品質評価。トリガー精度・ステップ明確度・エラー処理・ドキュメント品質を静的に採点する。

🔗 coherence

構造的整合性を4軸（規則性・一貫性・完全性・簡潔性）で評価する。スキル間の依存関係矛盾を検出する。

📈 telemetry

テレメトリ3軸（使用頻度・成功率・ユーザー修正率）で実際の動作実績を評価する。使われていないスキルを特定できる。

⚖️ constitutional

原則ベース LLM Judge 評価 + /cso security 軸。安全性・倫理性・セキュリティを包括的に審査する。

💥 chaos

仮想除去ロバストネス評価。スキルを一つずつ仮想的に除去してシステム全体への影響を測定する。依存関係の脆弱性を発見する。

🌍 environment

coherence + telemetry + constitutional + skill_quality を動的重みで統合した総合スコア。重み: coherence 0.25 / telemetry 0.45 / constitutional 0.30。

🔌 plugin

rl-anything プラグイン自身の統合 fitness。プラグインの自己評価に使用し、メタレベルの品質を計測する。

⚠️ environment fitness の重み: coherence 0.25 / telemetry 0.45 / constitutional 0.30。audit --coherence-score --telemetry-score --constitutional-score で各軸のスコアを個別に確認できます。

カスタム fitness の追加

$ cat scripts/rl/fitness/my_domain.py

def score(skill_content: str, context: dict) -> float:
    """0.0 〜 1.0 のスコアを返す"""
    return 0.8

# 使用する
/rl-anything:optimize my-skill --fitness my_domain

クイックスタート

あなたとClaude Codeが毎日繰り返すやり取りが、スキルを育てていきます。
ここでは「導入初日」「毎日の運用」「困ったとき」という3つの場面でストーリーを追いながら、主要コマンドを紹介します。

🌱 Day 1 — まず土台を作る

インストール直後は、過去のセッション履歴が蓄積されていません。backfill を実行して既存の作業ログを取り込み、rl-anything に「あなたのプロジェクトの素性」を教えましょう。

1
過去ログを取り込む
/rl-anything:backfill

既存セッション履歴を解析し、よく使うパターン・修正傾向を学習します。初回のみ。

↓
2
仕様書を初期化する
/rl-anything:spec-keeper init

コードベースを読んで SPEC.md を生成します。次セッションへの引き継ぎ書の土台になります。

↓
3
健康診断で現状を把握する
/rl-anything:audit

coherence / telemetry / constitutional の3軸でスコアを算出し、改善候補を一覧表示します。

Claude との作業中にフックが自動で観測を続け、セッション終了ごとにスキルを磨いていきます。あなたがすることは1コマンドだけです。

🎭 シナリオ別ガイド

🤔 「いつもClaude が同じミスをする」

修正パターンを corrections.jsonl に記録し、スキルとルールに反映します。

/rl-anything:reflect

何度もあなたが直した操作がパターン認識され、スキルの pitfall として自動追加されます。

🎯 「このスキルだけ精度を上げたい」

全体 evolve を待たずにピンポイントで特定スキルを改善します。

/rl-anything:optimize my-skill
# または自己進化パターンを組み込む
/rl-anything:evolve-skill my-skill
🚧 「なかなか解決しない問題がある」

行き詰まりを診断し、戦略提案 → Agent 起動まで一気に行います。

/rl-anything:breakthrough <問題の説明>

または独立した視点でのレビューが欲しいときは /rl-anything:second-opinion

🗂 「複数プロジェクトの状態を把握したい」

fleet コマンドで全プロジェクトを横断的に監視します。

$ bin/rl-fleet status
$ bin/rl-fleet tokens
$ bin/rl-fleet test-guard status

env_score が低いプロジェクトには、表示されたコマンドをそのまま実行して改善できます。

📋 「次のセッションに引き継ぎたい」

作業文脈は checkpoint 機構が自動保存し、次セッション開始時に自動復元します。手動操作は不要です。仕様書を最新化したい場合のみ実行します。

# 作業文脈は自動復元（操作不要）。仕様書を最新化する場合のみ:
/rl-anything:spec-keeper update
🧹 「PRマージ後に後片付けしたい」

ブランチ・worktree・tmp ディレクトリ・未クローズ Issue などの痕跡を候補提示しながら削除します。

/rl-anything:cleanup

個別承認方式なので、消したくないものは対話で除外できます。

📊 使い続けるとどう変わるか

rl-anything は env_score（0.0〜1.0）をレベルに変換して表示します。audit 後のグリーティングで現在のレベルと称号が確認できます。

Lv 1–3
初期セットアップ完了。スキルが素朴な状態。
Lv 4–6
修正フィードバックが蓄積され、pitfall が自動追加されはじめる。
Lv 7–9
スキルが文脈を理解し、同じミスをほぼしなくなる。
Lv 10
🏆 Constitutional / Telemetry / Coherence 全軸高スコア。自律運用の完成形。

アーキテクチャ

rl-anything の内部構造です。すべてのコンポーネントは独立して動作し、単一の plugin.json マニフェストで統合されています。
