rl-anything

スキル・ルールの 自律進化パイプライン、修正フィードバックループ、直接パッチ最適化、fleet 観測・介入を提供する Claude Code プラグイン。

Claude Code Plugin Observe hooks 15個 適応度関数 8種 自律進化
💡 rl-anything は LLM コストゼロの hooks 群で動作を観測し、corrections.jsonl に蓄積されたフィードバックをもとに CLAUDE.md・rules・skills を自動改善します。

インストール

$ claude plugin install rl-anything

インストール後、既存セッション履歴を取り込む初回セットアップを実行します。

# 初回セットアップ
/rl-anything:backfill

動作環境

要件詳細
Claude Codev2.1.83 以上（userConfig manifest 対応）
Python3.9+（scripts/ 実行時）
DuckDBトークン消費 DB に使用（optional だが推奨）
OSmacOS / Linux

4つの柱

rl-anything のすべての機能は4つの設計原則に基づいています。

🔄

自律進化

evolve · discover · reorganize · prune · audit

Observe → Diagnose → Compile → Housekeeping → Report の3ステージパイプラインでスキル・ルールを自動改善する。

🔁

フィードバックループ

reflect

ユーザーの修正パターンを検出し、corrections.jsonl に蓄積。CLAUDE.md・rules に構造化して反映する。

⚡

直接パッチ最適化

optimize · rl-loop · generate-fitness · evolve-fitness

corrections/context をもとに LLM 1パスでパッチを生成し、regression gate で品質を保証する。

🌐

Fleet 観測・介入

fleet (bin/rl-fleet)

全プロジェクト横断で env_score・導入状況を一覧表示。低スコアのプロジェクトに直接介入できる。

設計思想

🔬 Frozen Executor + Trainable Curator — Claude Code 本体（executor）は frozen と扱い、rl-anything はスキル群と curation ロジック（curator）のみを進化対象とします。SkillOS 論文（arXiv:2605.06614）が同設計で cross-backbone 汎化を実証しており、ADR-023 として明文化されています。
