## Context

prune SKILL.md の Step 3（人間承認フロー）は現在2段階:
1. Stage 1: 全候補一覧をテキスト表示
2. Stage 2: AskUserQuestion で一括方針選択（全てアーカイブ / 個別に選択 / スキップ）
3. Stage 3: 「個別に選択」の場合のみ各スキルごとに AskUserQuestion

問題: 一括方針選択が冗長で、推薦ラベルだけでは判断根拠が不足。

## Goals / Non-Goals

**Goals:**
- スキルごとに「削除すべきか」の分析・理由を提示した上でユーザーに聞く
- 一括方針選択（Stage 2）を廃止し、最初から個別レビューに入る
- 候補が多い場合のショートカット（残り全てアーカイブ / 残り全てスキップ）

**Non-Goals:**
- prune.py の候補検出ロジック変更
- 新しい Python コードの追加（SKILL.md のフロー定義のみ）

## Decisions

### Decision 1: Stage 2（一括方針選択）を廃止し、個別レビューをデフォルトに

**理由**: ユーザーは各スキルの判断根拠を見てから決めたい。一括選択は「とりあえずスキップ」になりがちで、prune の価値を損なう。

**代替案**: Stage 2 を残して「個別に選択」をデフォルトにする → 不要な1ステップが増えるだけなので廃止の方がシンプル。

### Decision 2: 各スキルの分析をテキスト出力 → AskUserQuestion の順で

**形式**:
1. テキスト出力: スキル名、description、推薦ラベル、**判断理由**（2-3行）
2. AskUserQuestion: アーカイブ / 維持 / 後で判断（1-2件目）、アーカイブ / 維持 / 残り全てスキップ（3件目以降）

**判断理由の観点**:
- なぜ使われていないか（用途が限定的、季節性がある等）
- 今後使われる可能性（汎用性、トリガー数）
- 他スキルとの重複・統合状況
- 参照/テンプレート価値

### Decision 3: ショートカット選択肢は3件目以降に表示

候補が2件以下なら「残り全て〜」は不要。3件目以降の AskUserQuestion に追加する。

**選択肢構成**:
- 候補 1-2件目: `アーカイブ` / `維持` / `後で判断`（3択）
- 候補 3件目以降: `アーカイブ` / `維持` / `残り全てスキップ`（3択、options 上限4つのルールに収まる）

「残り全てアーカイブ」は危険なので選択肢に入れず、テキスト入力で明示的に依頼する形とする。

## Risks / Trade-offs

- [候補が10件以上ある場合、1つずつ確認が面倒] → 「残り全てスキップ」ショートカットで緩和。また prune 候補は通常数件程度
- [分析の精度がスキルの SKILL.md の充実度に依存] → description 空文字時のフォールバック（既存 Step 2 ルール）で対応済み
