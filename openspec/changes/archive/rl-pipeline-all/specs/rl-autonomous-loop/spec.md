## ADDED Requirements

### Requirement: 自律進化ループ
ベースライン取得→バリエーション生成→評価→選択→人間確認の1サイクルを自動化するオーケストレーターを提供する。

#### Scenario: dry-run ループ実行
- **WHEN** `--target SKILL.md --dry-run` で実行する
- **THEN** ダミースコアでフルループが完走し、結果が出力ディレクトリに保存される

#### Scenario: 複数ループ実行
- **WHEN** `--loops 3` で実行する
- **THEN** 3回のループが順次実行され、各ループの結果が記録される

#### Scenario: スコア改善なし
- **WHEN** 最良バリエーションのスコアがベースライン以下である
- **THEN** 自動的にスキップし、元のスキルを維持する

### Requirement: 人間確認ステップ
デフォルトでは最良バリエーションの適用前に人間の承認を求める。

#### Scenario: 人間確認（承認）
- **WHEN** 最良バリエーションが提示され、人間が承認する
- **THEN** スキルファイルに最良バリエーションが適用される

#### Scenario: 人間確認（却下）
- **WHEN** 最良バリエーションが提示され、人間が却下する
- **THEN** 元のスキルを維持し、次のループに進む

#### Scenario: 自動承認モード
- **WHEN** `--auto` オプションで実行し、スコアが改善されている
- **THEN** 人間確認をスキップし自動で適用する

### Requirement: 履歴記録
全ループの結果を JSONL 形式で蓄積する。

#### Scenario: 履歴ファイル更新
- **WHEN** 1ループが完了する
- **THEN** 出力ディレクトリの `history.jsonl` にループ結果が追記される

#### Scenario: ラン別結果保存
- **WHEN** 全ループが完了する
- **THEN** 出力ディレクトリの `{run_id}/result.json` に全ループの結果が保存される
